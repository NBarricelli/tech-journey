# **AZ-900 Notes**

## **Azure Architecture & Services**

### **Azure Virtual Machines**
- Virtual environments that can be deployed and managed quickly and easily for compute, storage, servers, etc.
- Common use cases include testing & development, cloud app hosting, disaster recovery, and "lift and shift" migration.
- Grouped into families so you can choose a size based on workload needs.

When **provisioning VMs**, you choose the size (CPU cores, RAM), storage disks, and networking (IP addresses, port config, etc.). With Azure VMs, at a fundamental level, you start by selecting a family that matches your workload behavior. Then, you choose the size based on performance needs and scale as needed.

**Virtual Machine Scale Sets** allow you to create and manage groups of identical, load-balanced VMs. This ensures you do not have to manually keep VM config consistent and micromanage utilization.
**Virtual Machine Availability Sets** improve VM resiliency within a region and reduce the chance that all VMs are affected by maintenance or disaster. Availability Sets group VMs by **Update Domain**, which are VMs that can be rebooted together during planned maintenance, and **Fault Domain**, which are VMs that share a potential power or network failure point.
